## 評価指標の再設計（最優先）

教授陣から「4軸が抽象的すぎて信頼性がない」との指摘を受け、具体化のプロセスが必要。

### TASK-EVAL-1: 聞きたい評価観点を具体レベルで列挙
- [ ] 対話中に評価したい具体的な観点をフラットに列挙する（抽象軸から考えない）
  - 例: 相槌が自然か / ターンテイキングのタイミング / 発音の明瞭さ / ポーズの妥当性 / 内容の噛み合い / ...
- [ ] 参考: 既存の対話音声の主観評価プロトコル（藤井先生が言及したテキスト対話の主観評価手法）をリサーチ
- [ ] 音声合成の主観評価との違いを整理（合成は静的、対話は動的で評価項目が多い）

### TASK-EVAL-2: 具体項目を抽象軸に整理
- [ ] TASK-EVAL-1 で出した具体項目を分類し、どう抽象軸にまとめるか検討
- [ ] 「具体 → 抽象」の順序で導出したことを説明できる形にする
- [ ] 現状の4軸（音声自然性/聴感自然性/意味理解性/対話有用性）との対応をつける

### TASK-EVAL-3: 指標粒度の整合性を担保
- [ ] 「音声・聴感・意味」の3軸と「対話有用性」4つ目の粒度差を解消
  - 指摘: 最初の3軸は「低品質前提」、対話有用性は1段高いレベル
- [ ] 「音も悪い・会話できていない、しかし有用」のような矛盾パターンを生まない設計

### TASK-EVAL-4: リファレンス音声の検討
- [ ] 「この音声は音声自然性が高い例」といった基準音声を用意するか判断
- [ ] 用意する場合、各指標について high/low のサンプルを選定

### TASK-EVAL-5: 被験者制約の方針決定
- [ ] クラウドワーカー（不特定多数）vs 専門家（ブリザードチャレンジ方式）
- [ ] 学内限定 or 学外公開かも含めて決定
- [ ] 決めた方針が信頼性にどう効くか論文に書ける形にする

### TASK-EVAL-6: 当事者 vs 第三者評価の整理
- [ ] 当事者評価しかできない構造上の制約を論点として明示
- [ ] 当事者は採点が甘くなる傾向を、どう補正 or 議論するか検討
- [ ] MOS 参考で第三者視点を入れていた根拠をソース付きで整理

### TASK-EVAL-7: 会話内容の発散対策
- [ ] A と B で会話が発散した場合の AB 比較の有効性低下を検討
- [ ] 現状「各システム2回ずつ対話」で吸収できる想定だが、検証が必要
- [ ] 必要ならインストラクション（誘導しすぎない範囲の枠）を入れる
