# Speech Arena V2 スコープ

### 評価を止めずに UI を書き換える — 並走開発の設計

<p>2026-04-23 / 阿部</p>

---

# 背景と課題

<div class="diff">
<div class="col v1">
<div class="label">現状 (V1)</div>
<div class="body">

- ホーム画面が **4 行の箇条書き**で簡素
- 待ち時間がないため **操作説明が読まれない**
- 会話は `window.open` で **別タブ起動** → 戻り忘れ
- 評価パネルは会話後に表示 → 軸が頭に入らない

</div>
</div>
<div class="col v2">
<div class="label">目指す姿 (V2)</div>
<div class="body">

- PoC のような **bento-card ランディング**
- Protocol 01/02/03 ステップを **常時表示**
- `protocol.ts` 直結で **同タブ埋め込み**
- 右サイドに **評価パネル常置** → 較正効果

</div>
</div>
</div>

<div class="card warn">

**制約**: SLT パイロット (5/15) の評価は現行で進行中。**止められない**。

</div>

---

# 設計方針: 別リポで並走

<div class="card">

<span class="pill v1">V1</span> `kobas-lab/speech-arena` — **走らせたまま**（評価収集）
<span class="pill v2">V2</span> `abePclWaseda/speech-arena-next` — **UI 書き換えと音声埋め込み**

</div>

<div class="card ok">

**メリット**
- V1 を壊すリスクがゼロ（DB・デプロイ・ドメイン独立）
- V2 は自由に実験できる（PR レビュー待ちなし）
- 成功基準を満たしたら main に取り込む／据え替える判断を後から

</div>

---

# 変える / 継承する / 別立てする

| 領域 | V1 → V2 |
|---|---|
| ホーム画面 | 4 行リスト → **bento-card + Protocol 常時表示** |
| 音声対話 | `window.open` 新タブ → **protocol.ts で同タブ埋め込み** |
| 評価パネル | 会話後に順次 → **右サイドに常置**（軸リマインダー） |
| 評価軸 | Q1〜Q4 現行版 → **Q3 再設計版** (4/28 確定) |
| オンボーディング | 説明未読 → **練習トライアル / 参照クリップ**検討 |
| 見た目 | shadcn デフォルト | dark mode + framer-motion |

**継承**: Prisma スキーマ・API 契約・GPU セッション抽象化・Moshi Tunnel スクリプト・Leaderboard

---

# 別立てが必須なもの（事故防止）

| 項目 | 理由 |
|---|---|
| **Supabase プロジェクト** | <span class="warn">`prisma migrate` が V1 の評価 DB を壊すと致命的</span> |
| Vercel プロジェクト | デプロイ独立、preview URL で実験しやすい |
| GitHub リポ | `abePclWaseda/speech-arena-next` で権限と PR フロー独立 |
| `.env` | `DATABASE_URL`, `HF_TOKEN` 等を分離 |

<div class="card ok">

**共用するもの**: Moshi サーバー（RTX 3090 × 2 を V1 が占有中）— V2 側は Tunnel URL を `.env` で参照するだけ

</div>

---

</blockquote>

# 成功基準

- V1 と同じ試行フローが完走する（2 モデル × 2 セッション + 投票）
- <span class="ok">会話がタブ切り替えなしで完結する</span>
- <span class="ok">評価パネルが会話中から見えている</span>
- <span class="ok">初見ユーザーが説明に触れる動線がある</span>

これらを満たせば、V1 からの移行判断（据え替え or 取り込み）に進める。
